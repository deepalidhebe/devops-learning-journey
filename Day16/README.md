# 🚀 Day 16 – AWS Services for DevOps Engineers

## 📌 Overview
This project documents my learning journey into **AWS services most relevant for DevOps engineers**.  
AWS offers 200+ services, but as a DevOps engineer, focusing on automation, efficiency, and security is key.  
This repo highlights the core services I explored and why they matter.

---

## 🛠️ Key AWS Services for DevOps
1. **EC2 (Elastic Compute Cloud)**  
   - Virtual servers for running applications.  
   - Foundation for deployments and scaling workloads.

2. **VPC (Virtual Private Cloud)**  
   - Secure networking environment.  
   - Includes subnets, CIDR blocks, security groups, inbound/outbound rules.

3. **EBS (Elastic Block Store)**  
   - Persistent storage volumes for EC2.  
   - Supports snapshots, backups, and re‑attachments.

4. **S3 (Simple Storage Service)**  
   - Scalable, cost‑effective object storage.  
   - Now encrypted by default for security.

5. **IAM (Identity & Access Management)**  
   - Role‑based access control.  
   - Ensures least privilege and secure user policies.

6. **CloudWatch**  
   - Monitoring and observability.  
   - Tracks metrics, logs, and triggers alerts.

7. **Lambda**  
   - Serverless compute for short actions.  
   - Automates tasks like encrypting unsecure EBS volumes.

8. **CI/CD Services**  
   - **CodePipeline** – workflow orchestration.  
   - **CodeBuild** – compile, test, package.  
   - **CodeDeploy** – deploy artifacts to EC2/on‑premises.

9. **AWS Config**  
   - Tracks resource configurations.  
   - Ensures compliance and governance.

10. **Billing & Cost Management**  
    - Visibility into spend across services.  
    - Helps optimize resource usage.

11. **KMS (Key Management Service)**  
    - Secure storage and rotation of keys/certificates.  
    - Enables encryption for S3, EBS, and other services.

12. **CloudTrail**  
    - Auditing and governance.  
    - Records API calls and user activity for risk management.

---

## 🌍 Key Learning
- DevOps on AWS is about **automation + efficiency + security**.  
- Not every AWS service is required - focus on the ones that directly support CI/CD, monitoring, networking, and secure deployments.  
- Combining services (e.g., CloudWatch + Lambda) enables proactive guardrails and compliance enforcement.\
