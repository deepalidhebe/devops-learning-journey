# 📌 Day 21 - Project Management Tools for DevOps Engineers

### 🔑 Key Takeaways
- **Agile Methodology**  
  - Contrast with **Waterfall**: Agile delivers in small increments (Sprints) instead of long cycles.  
  - Benefits: Faster feedback, reduced risk, continuous improvement.  
  - Common practices: Sprint planning, retrospectives, backlog grooming.

- **Jira**  
  - Tool to implement Agile project management.  
  - Tracks epics, stories, bugs, and team velocity.  
  - Free trial available via Atlassian - easy to practice at home.

- **Confluence & SharePoint**  
  - Knowledge‑sharing platforms for internal documentation.  
  - Ensure smooth knowledge transfer when engineers leave or new members join.  
  - Confluence integrates seamlessly with Jira.

- **ServiceNow**  
  - Used for **Incident Management** (tracking production issues) and **Change Management** (controlled deployments and fixes).  
  - Integrates with monitoring systems to auto‑create incidents.

- **ReadTheDocs**  
  - Open‑source alternative to Confluence/SharePoint.  
  - Integrates with GitHub for automated documentation hosting.

- **GitHub / Azure Boards**  
  - Open‑source projects often use GitHub Projects or Azure Boards for task tracking.  
  - Supports ticket lifecycle: To‑Do → In Progress → Review → Done.

### 🧩 Why This Matters
DevOps engineers aren’t just about CI/CD pipelines, they must also **navigate organizational workflows**. 
Knowing these tools ensures you can:  
- Communicate effectively with teams.  
- Track and document progress.  
- Handle incidents and changes professionally.  
- Contribute to open‑source projects with proper documentation.
